Day 4 - Generate reports in HTML and Allure

Solution Guide Example: API Test Framework

This document explains the structure and functionality of the test framework located in the /aula4/solucao/ directory. This framework is the result of completing the exercises from Day 4.

Framework Structure

The framework is organized into a simple and scalable structure:

/aula4/solucao/
├── conftest.py
├── pytest.ini
├── tests/
│   ├── __init__.py
│   ├── test_github_api.py
│   └── test_jsonplaceholder_api.py
└── test_run.log (generated after a test run)

File Breakdown

conftest.py

This is the most important file for configuring pytest. It acts as a local plugin repository for our framework.

  • `setup_logging()` and `pytest_sessionstart`: These work together to configure logging (to console and `test_run.log`) once at the very beginning of the test session.
  • `ApiClient` class: A custom class that inherits from `requests.Session`. Its main purpose is to "remember" the last API response by storing it in `self.last_response`. This is crucial for our reporting hook.
  • `api_client` fixture: A session-scoped fixture that creates and provides a single instance of our `ApiClient` to all tests. This is efficient and allows us to share state (like the last response) across the test session if needed.
  • `pytest_runtest_makereport` hook: This powerful hook inspects the result of each test. If a test fails, it checks if the `api_client` was used, retrieves the `last_response` from it, and injects the response body directly into the HTML report. This makes debugging assertion errors much faster.

pytest.ini

This file provides default command-line arguments for pytest, so you don't have to type them every time.

  • `log_cli = true`: Enables live logging to the console during the test run.
  • `log_cli_level = INFO`: Sets the default verbosity of the console logs to `INFO` and higher (INFO, WARNING, ERROR, CRITICAL).

tests/ directory

This directory contains all our test suites, organized by the API they target.

  • `__init__.py`: An empty file that tells Python to treat the `tests` directory as a package. This helps with imports and test discovery.
  • `test_github_api.py` & `test_jsonplaceholder_api.py`: These files contain the actual test cases, refactored to use the framework's features. Notice they no longer import `requests` directly but instead receive the `api_client` fixture. They also use `logging` for detailed output and `@allure` decorators for rich reporting.

How to Run the Tests

To execute the tests, navigate to the framework's root directory in your terminal:

cd /path/to/your/project/aula4/solucao/

Basic Test Run

Simply run `pytest`. Thanks to `pytest.ini`, this will execute all tests and show `INFO` level logs in your console.

pytest

Running with a Different Log Level

To see more detailed logs (e.g., `DEBUG`), you can override the setting from `pytest.ini` on the command line:

pytest -o log_cli_level=DEBUG

Generating the HTML Report

To generate a self-contained HTML report, use the `--html` flag. This is useful for a quick, shareable summary.

pytest --html=report.html

After running, a `report.html` file will be created. Open it in your browser. If any test fails, you will see the "API Response Body" section thanks to our hook.

Generating and Viewing the Allure Report

Allure provides a much richer, interactive report. This is a multi-step process.

Prerequisite: Install Allure Commandline

The `allure-pytest` plugin only creates the result files. To generate and view the HTML report, you need the Allure command-line tool. If you see an error like zsh: command not found: allure, you need to install it.

On macOS with Homebrew (Recommended):

brew install allure

For other operating systems, please refer to the official Allure documentation.

Step 1: Generate Allure result files

This command runs the tests and saves the Allure report data (in JSON format) to a directory.

pytest --alluredir=./allure-results

Step 2: Serve the Allure report

This command uses the Allure command-line tool to generate the HTML report from the JSON files and open it in your browser.

allure serve ./allure-results

You can now explore the interactive report, see the detailed steps for each test, and view any attachments you added (like the JSON response bodies).
